- (decision) UX scope: start with public menu (5 screens, UJ-3 Jean). Sources: PRD + PRFAQ + Brief.
- (decision) Identification: navigation libre sans identification -> téléphone requis à la commande (option c).
- (decision) Client inconnu: commande via QR sans wallet, paiement au comptoir. Téléphone saisi -> fidélité passive.
- (decision) Order types: QR table -> table+menu, QR comptoir -> takeaway. Même flux, order_type: table|takeaway.
- (decision) Ton: utilitaire chaleureux. Pas d'images. Catégories claires, prix FCFA visibles, typo 4'. Collapsables pour persos.
- (decision) WiFi URL: IP fixe sur QR (192.168.1.42:PORT/menu/4). IP affichée au démarrage. Panneau physique dans l'établissement. Pas d'écran d'erreur réseau.
- (decision) Notification prête: workflow humain (serveur/caissier appelle ou clochette cuisine). Statut en ligne = confort visuel, pas de polling.
- (decision) Langue: i18n-ready dès V1, clés dans templates. Contenu FR V1, EN en P1.
- (decision) Appel serveur: V2. V1 = humain (voix haute).
- (decision) Form factor: mobile 4' pour client QR, desktop pour caisse/cuisine.
- (decision) Working mode: Fast Path. Pas d'images, pas d'animations, pas de WebSocket.
- (change) EXPERIENCE.md: Flow 1 — climax modifié. Ajout étape 8 (serveur appelle Jean physiquement).
- (change) DESIGN.md: prix passe en text-primary. Tap feedback passe en scale(0.97) 150ms.
- (decision) Assumptions confirmées : Nuxt UI v4, green primary, zinc neutral, Inter, sticky footer, pas d'images/icônes.
- (decision) Note : prix en vert = confusion avec CTA. À tester en vrai, pas corrigé maintenant.
