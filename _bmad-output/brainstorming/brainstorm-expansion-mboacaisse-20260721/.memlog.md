# Technique 1 — Le Client Roi

- (idea by user) Pré-commande client : menu public page non-auth servie par Axum
- (idea by user) QR code par table (URL encode table ID, génération serveur, impression papier)
- (idea by user) Paiement client via Mobile Money (MTN MoMo, Orange Money) — routeur paiement + webhook + statut transaction
- (idea by user) Programme fidélité : points, cagnotte, crédit client
- (idea by user) Cycle commande enrichi : pending_payment → paid_preparing → ready → delivered
- (idea by user) Table management : assignation client→table→commande
- (idea by coach) Paiement = point d'inflexion : MoMo débloque dépôt client, pré-commande, reconnaissance sans carte. Défi offline.
- (insight by user) Architecture wallet : wallet client = source de vérité en SQLite locale. MoMo/cash = sources d'approvisionnement. Paiement toujours depuis wallet.
- (insight by user) Wallet multi-sources : Cash, MoMo, Gift, Cashback, Transfer. Solde jamais stocké (SUM(amount) WHERE client_id). Pas de dérive.
- (idea by user) Client sans téléphone : ID interne CLI-XXXX, wallet rattaché, pas de pré-commande mais reste fonctionnel
- (idea by user) Menu public : 5 écrans (landing QR, menu, panier, identification téléphone, confirmation). Paiement avant validation commande.
- (decision by user) Paiement avant validation commande : wallet check immédiat, sinon message solde insuffisant. Wallet négatif = choix admin pas défaut.
- (decision by user) QR par table (pas sélection manuelle) + fallback admin pour saisie manuelle numéro table
- (idea by user) Cashback auto 5% sur chaque commande payée par wallet = levier adoption, zéro config gérant
- (idea by coach) Seuil cashback progressif : 3%→5%→8% selon nb commandes. Game design sans UI.
- (idea by user) Parrainage : numéro recommandé lié à l'inscription, 1000FCFA sur deux wallets
- (decision by user) Dépôt de garantie : feature optionnelle désactivée par défaut + avertissement setup (zone grise régulation financière)

# Technique 3 — Le Bug du Succès

- (insight by coach) Wallet par instance = pas de sync groupe. Client a solde différent dans chaque établissement. OK pour 3 ans.
- (idea by user) mDNS : nom personnalisable au setup (chezbob.local) au lieu de mboacaisse. Fallback admin.
- (idea by coach) wallet_ledger : table append-only (INSERT-only, jamais UPDATE/DELETE) pour chaque transaction wallet. Backup toutes les 5min.
- (idea by coach) Impression queue async avec retry + fallback ticket numérique (écran client) quand imprimante hors service
- (decision by user) Tier architecture : P0 wallet_ledger + P0 impression queue / P1 mDNS personnalisable / P2 sync groupe
- (idea by user) Migration wallet_ledger rétro-compatible : créer ledger + rejouer historique commandes payées. Sinon ledger vide = crash mortel.
